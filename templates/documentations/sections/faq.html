Questions Fréquentes

Réponses aux questions les plus courantes sur PW-Analyst

Questions Générales

PW-Analyst est une plateforme centralisée de gestion et d'analyse des tests automatisés. Elle permet de :

  • Collecter automatiquement les résultats de vos tests depuis vos pipelines CI/CD
  • Organiser vos tests par projets et catégories avec un système de tags
  • Analyser les tendances de qualité avec des dashboards et rapports détaillés
  • Gérer les permissions d'équipe avec des rôles hiérarchiques (Viewer, Manager, Admin)
  • Identifier les tests instables (flaky tests) et les régressions

Prérequis minimum :

  • Python 3.9 ou supérieur
  • Base de données (SQLite pour dev, PostgreSQL recommandé pour production)
  • 512 MB RAM minimum (2 GB recommandé)
  • 200 MB d'espace disque

Optionnel pour de meilleures performances :

  • Redis pour le cache et les tâches asynchrones
  • Serveur web (Nginx/Apache) + Gunicorn pour la production
  • Certificat SSL pour HTTPS

Plusieurs méthodes d'import sont disponibles :

1. Import via API REST :
curl -X POST /api/results/import/ -H "Authorization: ApiKey your-key" -d @results.json
2. Formats supportés :
  • JSON natif PW-Analyst
  • JUnit XML
  • TestNG XML
  • pytest JSON
3. Interface web : Upload manuel de fichiers via l'interface d'administration (pour les Managers/Admins)

Permissions et Groupes

👷 MANAGER :

  • Toutes les permissions Viewer
  • Création/modification des tests
  • Import de résultats de tests
  • Gestion des tags (couleurs, etc.)
  • Modification des statuts de tests
  • ✗ Pas de gestion des projets
  • ✗ Pas d'accès administration

👑 ADMIN :

  • Toutes les permissions Manager
  • ✓ Création/modification/suppression des projets
  • ✓ Accès interface d'administration Django
  • ✓ Gestion des utilisateurs et groupes
  • ✓ Configuration système complète

Non, un utilisateur ne peut appartenir qu'à un seul groupe à la fois par conception. Cela simplifie la gestion des permissions et évite les conflits.

💡 Astuce : Les groupes sont hiérarchiques. Un Admin hérite automatiquement de toutes les permissions Manager et Viewer. Il n'est donc pas nécessaire d'assigner plusieurs groupes.

  1. Connectez-vous avec un compte Manager ou Admin (les Viewers n'ont pas accès aux clés API)
  2. Allez dans votre Profil utilisateur (icône en haut à droite)
  3. Cliquez sur "Gestion des clés API"
  4. Cliquez sur "Générer nouvelle clé"
  5. Copiez immédiatement la clé - elle ne sera plus affichée par la suite
  6. Configurez la clé dans vos variables d'environnement CI/CD

⚠️ Sécurité : La clé hérite des permissions de votre compte. Ne la partagez jamais et révoquez-la si elle est compromise.

Questions Techniques

✅ RECOMMANDÉ :

  • PostgreSQL 12+ - Excellent pour production
  • Meilleures performances pour les requêtes complexes
  • Support natif des types JSON
  • Transactions ACID complètes
  • Backup et réplication avancés

⚠️ ALTERNATIFS :

  • MySQL 8.0+ - Acceptable si déjà utilisé
  • SQLite - Uniquement pour développement/test

❌ Éviter : SQLite en production (problèmes de concurrence avec plusieurs utilisateurs)

Configuration recommandée avec Nginx : 

# /etc/nginx/sites-available/pw-analyst
server {
    listen 80;
    server_name votre-domaine.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name votre-domaine.com;

    ssl_certificate /path/to/certificate.crt;
    ssl_certificate_key /path/to/private.key;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto https;
    }
}

Options pour obtenir un certificat SSL :

  • Let's Encrypt (gratuit) : certbot --nginx -d votre-domaine.com
  • Certificat payant (Comodo, DigiCert, etc.)
  • Wildcard certificate pour plusieurs sous-domaines

1. Sauvegarde de la base de données :

pg_dump pw_analyst > backup_$(date +%Y%m%d).sql

2. Sauvegarde des fichiers statiques :

tar -czf media_backup.tar.gz /path/to/pw-analyst/media/

3. Script de sauvegarde automatique (crontab) :

# Sauvegarde quotidienne à 2h du matin 0 2 * * * /usr/local/bin/backup-pw-analyst.sh

💡 Conseil : Testez régulièrement vos sauvegardes en restaurant sur un environnement de test.

Dépannage

Causes possibles et solutions :

1

Clé API invalide ou expirée

→ Régénérez une nouvelle clé API dans votre profil

2

Permissions insuffisantes

→ Seuls les Managers et Admins peuvent importer des résultats

3

Format du header incorrect

→ Utilisez : Authorization: ApiKey votre-cle

Optimisations recommandées :

🚀 Performance

  • Configurer Redis pour le cache
  • Utiliser PostgreSQL au lieu de SQLite
  • Activer la compression gzip dans Nginx
  • Servir les fichiers statiques via CDN
  • Optimiser les index de base de données

⚙️ Configuration

  • Augmenter les workers Gunicorn
  • Configurer le pool de connexions DB
  • Activer le cache des templates Django
  • Utiliser Celery pour les tâches lourdes
  • Monitorer avec des outils comme Sentry
