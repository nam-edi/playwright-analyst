Groupes et Permissions

Comprendre les différents niveaux d'accès dans Playwright Analyst

Vue d'ensemble

Playwright Analyst utilise un système de groupes pour gérer les permissions des utilisateurs. Chaque utilisateur peut appartenir à un ou plusieurs groupes, et chaque groupe définit les actions autorisées dans l'application.

Groupe Admin

Accès complet à toutes les fonctionnalités

Permissions:

Accès à l'administration
Gestion des utilisateurs et groupes
Création/modification/suppression des projets
Import/modification des résultats de tests
Configuration des features et CI
Gestion des clés API

Groupe Manager

Gestion des tests et tags, lecture seule des projets

Permissions:

Lecture seule des projets (pas de création/modification/suppression)
Import/modification des résultats de tests
Configuration des features projets
Modification des statuts de tests
Ajout/modification des commentaires
Pas d'accès à l'administration

Groupe Viewer

Accès en lecture seule uniquement

Permissions:

Consultation des projets
Consultation des tests et résultats
Consultation des exécutions
Accès au dashboard et statistiques
Pas de création/modification/suppression
Pas d'import de résultats

Cumul des permissions

Un utilisateur peut appartenir à plusieurs groupes simultanément. Dans ce cas, il aura toutes les permissions de tous ses groupes. Par exemple, un utilisateur dans les groupes "Manager" et "Viewer" aura toutes les permissions du Manager.

Superutilisateurs

Les superutilisateurs ont automatiquement toutes les permissions, indépendamment de leurs groupes. Ils peuvent accéder à l'administration et effectuer toutes les actions dans l'application.

Comptes de démonstration

Pour tester les différents niveaux d'accès, vous pouvez utiliser ces comptes :

🔴 superadmin admin123 Admin
🟡 manager manager123 Manager
🟢 viewer viewer123 Viewer
