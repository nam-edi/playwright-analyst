Configuration Playwright

Optimisez vos rapports de tests en configurant les métadonnées Playwright

Métadonnées recommandées

Recommandation : Pour obtenir un maximum d'informations et de fonctionnalités dans vos rapports, il est fortement recommandé d'ajouter des métadonnées dans votre configuration Playwright.

Configuration dans playwright.config.ts

Ajoutez le bloc metadata dans votre defineConfig : 

import { defineConfig } from '@playwright/test';

export default defineConfig({
  // ... autres configurations ...

  metadata: {
    environment: process.env.NAMESPACE || 'local',
    name: process.env.NAME || 'woop-playwright',
    timestamp: new Date().toISOString(),
    ci: !!process.env.CI,
    buildHref: process.env.CI
        ? `${process.env.CI_SERVER_URL}/${process.env.CI_PROJECT_PATH}/-/pipelines/${process.env.CI_PIPELINE_ID}`
        : '',
  },

  // ... reste de la configuration ...
});

Description des métadonnées

environment

Identifie l'environnement d'exécution des tests (dev, staging, prod, etc.).

process.env.NAMESPACE || 'local'

name

Nom du projet ou de la suite de tests pour identification.

process.env.NAME || 'woop-playwright'

timestamp

Horodatage de génération du rapport pour traçabilité.

new Date().toISOString()

ci

Indique si les tests sont exécutés dans un environnement CI/CD.

!!process.env.CI

buildHref

Lien direct vers le build/pipeline CI pour un accès rapide depuis les rapports.

process.env.CI ? `${process.env.CI_SERVER_URL}/${process.env.CI_PROJECT_PATH}/-/pipelines/${process.env.CI_PIPELINE_ID}` : ''

Note : Cette configuration est adaptée pour GitLab CI. Adaptez les variables d'environnement selon votre plateforme CI (GitHub Actions, Jenkins, etc.).

Variables d'environnement courantes

GitLab CI

CI_SERVER_URL - URL du serveur GitLab
CI_PROJECT_PATH - Chemin du projet
CI_PIPELINE_ID - ID du pipeline
NAMESPACE - Variable personnalisée pour l'environnement
NAME - Variable personnalisée pour le nom du projet
